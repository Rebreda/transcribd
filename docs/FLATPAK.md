# Flatpak Build and CI

This project uses GitHub Actions for Flatpak builds and Flathub for end-user distribution.

## Manifests

| File | Purpose |
|------|---------|
| `io.github.rebreda.Transcribd.Devel.json` | Development builds (CI artifact) |
| `io.github.rebreda.Transcribd.json` | Production build (Flathub submission) |
| `flathub.json` | Flathub arch configuration |
| `generated-sources.json` | Offline npm dependency sources (auto-generated) |

## Build Flatpak Locally

1. Install prerequisites:

```bash
sudo dnf install -y flatpak flatpak-builder
```

2. Add Flathub remote:

```bash
flatpak --user remote-add --if-not-exists flathub https://flathub.org/repo/flathub.flatpakrepo
```

3. Install SDK/runtime and extensions:

```bash
flatpak --user install -y flathub \
  org.gnome.Sdk//48 \
  org.gnome.Platform//48 \
  org.freedesktop.Sdk.Extension.node20//24.08 \
  org.freedesktop.Sdk.Extension.typescript//24.08
```

4. Build repo and bundle (dev build):

```bash
flatpak-builder --user --force-clean --repo=repo flatpak-build io.github.rebreda.Transcribd.Devel.json
flatpak build-bundle repo transcribd-devel.flatpak io.github.rebreda.Transcribd.Devel
```

5. Install and run locally:

```bash
flatpak --user install -y ./transcribd-devel.flatpak
flatpak run io.github.rebreda.Transcribd.Devel
```

## GitHub Actions Workflow

Workflow file:
- `.github/workflows/flatpak.yml`

Triggers:
- Pull requests
- Pushes to `main`
- Manual dispatch

Output artifact:
- `transcribd-devel.flatpak`

## Release on Tag Push

The workflow publishes to GitHub Releases when you push a tag starting with `v`.

Example:

```bash
git tag v1.0.1
git push origin v1.0.1
```

GitHub Actions will:
1. Build `transcribd-devel.flatpak`.
2. Create a GitHub Release for the tag.
3. Attach `transcribd-devel.flatpak` to the release.

## npm Dependency Sources

Flathub builds have no network access. All npm packages must be declared as offline
sources in `generated-sources.json`. This file is auto-generated by the
`.github/workflows/update-npm-sources.yml` workflow whenever `package-lock.json` changes.

To regenerate it locally:

```bash
# Install the generator (requires Python 3.11+)
pip install "git+https://github.com/flatpak/flatpak-builder-tools.git#subdirectory=node"

# Remove node_modules first (required by the generator)
rm -rf node_modules

# Generate
flatpak-node-generator npm package-lock.json -o generated-sources.json
```

Commit `generated-sources.json` when it changes. The CI workflow does this automatically.

## Publishing to Flathub

Flathub is the primary Linux app store. Publishing there makes the app installable by anyone with:

```bash
flatpak install flathub io.github.rebreda.Transcribd
```

### Pre-submission checklist

Before opening the submission PR, verify:

- [ ] `generated-sources.json` is committed and up to date
- [ ] App builds and runs correctly from `io.github.rebreda.Transcribd.json`
- [ ] Metainfo passes validation: `appstream-util validate-relax --nonet data/io.github.rebreda.Transcribd.metainfo.xml.in.in`
- [ ] Desktop file is valid: `desktop-file-validate data/io.github.rebreda.Transcribd.desktop.in.in`
- [ ] App has a 256×256 PNG or SVG icon at `data/icons/io.github.rebreda.Transcribd.svg`
- [ ] Release tag `v1.0.0` is pushed and `io.github.rebreda.Transcribd.json` references it with the correct commit hash

Update the production manifest source to the actual tag and commit hash:

```json
"sources": [
  {
    "type": "git",
    "url": "https://github.com/rebreda/transcribd.git",
    "tag": "v1.0.0",
    "commit": "<output of: git rev-list -n1 v1.0.0>"
  },
  "generated-sources.json"
]
```

Run the linter locally:

```bash
flatpak install -y flathub org.flatpak.Builder
flatpak run --command=flatpak-builder-lint org.flatpak.Builder manifest io.github.rebreda.Transcribd.json
```

### Submission steps

1. Fork the Flathub repository:

```bash
gh repo fork --clone flathub/flathub
cd flathub
git checkout --track origin/new-pr
```

2. Create a submission branch:

```bash
git checkout -b add-transcribd new-pr
```

3. Create the app directory and copy the required files:

```bash
mkdir io.github.rebreda.Transcribd
cp /path/to/transcribd/io.github.rebreda.Transcribd.json io.github.rebreda.Transcribd/
cp /path/to/transcribd/generated-sources.json io.github.rebreda.Transcribd/
cp /path/to/transcribd/flathub.json io.github.rebreda.Transcribd/
```

4. Commit and push:

```bash
git add io.github.rebreda.Transcribd/
git commit -m "Add io.github.rebreda.Transcribd"
git push origin add-transcribd
```

5. Open a pull request against the **`new-pr`** base branch on GitHub (not `master`).
   Title: `Add io.github.rebreda.Transcribd`

> **Important:** Do not open the PR against `master`. Flathub reviews happen on the `new-pr` branch.

### After approval

Once the Flathub PR is approved and merged:
- A repository is created at `github.com/flathub/io.github.rebreda.Transcribd`
- You receive write access (requires 2FA on GitHub)
- Future releases are deployed by pushing to that repo — not through another submission PR
- See the [Flathub maintenance guide](https://docs.flathub.org/docs/for-app-authors/maintenance) for update workflows

## Troubleshooting

### Extension install issues

If extension installation fails in CI, pin the Freedesktop SDK version:

```bash
org.freedesktop.Sdk.Extension.node20//24.08
org.freedesktop.Sdk.Extension.typescript//24.08
```

### Manifest mismatch

If the workflow fails with an app ID or module mismatch:
1. Verify `"id"` in the manifest matches the expected app ID exactly.
2. Verify the module source URL and tag in the manifest.
3. Re-run the workflow after manifest changes.

### Build fails on stale state

```bash
rm -rf flatpak-build repo .flatpak-builder
```
